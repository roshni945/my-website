📖 Introduction

JavaScript ES6 (ECMAScript 2015) is one of the biggest updates to the JavaScript language. It introduced powerful features like let, const, Arrow Functions, Template Literals, Destructuring, Rest Parameters, and the Spread Operator. These features make JavaScript code more readable, cleaner, and easier to maintain.

In this tutorial, you'll learn every important ES6 concept with beginner-friendly explanations, practical examples, and interview-focused notes.

📖 What is ES6?

ES6 (ECMAScript 2015) is the sixth major version of JavaScript. It was released in 2015 and introduced many powerful features that make JavaScript code shorter, cleaner, and easier to write.

Before ES6, developers mostly used var for variables and traditional functions. ES6 introduced modern features such as let, const, Arrow Functions, Template Literals, Classes, Promises, and much more.

ES6 = ECMAScript 2015

📦 var vs let vs const

JavaScript provides three ways to declare variables.

Keyword Scope Can Reassign? Can Redeclare?
var Function Scope ✅ Yes ✅ Yes
let Block Scope ✅ Yes ❌ No
const Block Scope ❌ No ❌ No

Example using var

var name = "Roshni";

name = "Code Charm";

console.log(name);
Output:
Code Charm

Example using let

let city = "Mumbai";

city = "Delhi";

console.log(city);
Output:
Delhi

Example using const

const country = "India";

console.log(country);
Output:
India

💡 Best Practice

Use const by default. Use let only when the value needs to change. Avoid using var in modern JavaScript.

🌍 Block Scope

ES6 introduced Block Scope using let and const. A variable declared inside { } can only be accessed inside that block.

Example using let

{
    let age = 18;
    console.log(age);
}

console.log(age);
Output:
18
ReferenceError: age is not defined

Example using var

{
    var age = 18;
}

console.log(age);
Output:
18

💡 Quick Tip

var ignores block scope, while let and const respect block scope.

⬆️ Hoisting

Hoisting means JavaScript moves declarations to the top before executing the code.

Example with var

console.log(message);

var message = "Hello";
Output:
undefined

Example with let

console.log(message);

let message = "Hello";
Output:
ReferenceError

Example with const

console.log(pi);

const pi = 3.14;
Output:
ReferenceError

⚠ Common Mistake

Many beginners think let and const are not hoisted. Actually they are hoisted, but they remain in the Temporal Dead Zone (TDZ) until their declaration.

📝 Template Literals

Template literals use backticks (` `) instead of quotes. They allow variable interpolation and multi-line strings.

Without Template Literals

let name = "Roshni";

console.log("Hello " + name);
Output:
Hello Roshni

With Template Literals

let name = "Roshni";

console.log(`Hello ${name}`);
Output:
Hello Roshni

Multi-line String

let text = `JavaScript
ES6
Tutorial`;

console.log(text);
Output:
JavaScript
ES6
Tutorial

💡 Did You Know?

Template literals can directly evaluate JavaScript expressions.

Expression Example

let a = 20;

let b = 30;

console.log(`Sum = ${a + b}`);
Output:
Sum = 50

⚙️ Default Parameters

ES6 allows you to assign default values to function parameters. If no argument is passed, the default value is used automatically.

Without Default Parameter

function greet(name) {

    console.log("Hello " + name);

}

greet();
Output:
Hello undefined

With Default Parameter

function greet(name = "Guest") {

    console.log("Hello " + name);

}

greet();

greet("Roshni");
Output:
Hello Guest
Hello Roshni

💡 Tip

Default parameters help prevent undefined values and make functions easier to use.

🏹 Arrow Functions

Arrow Functions provide a shorter syntax for writing functions.

Normal Function

function add(a, b) {

    return a + b;

}

console.log(add(10, 20));
Output:
30

Arrow Function

const add = (a, b) => {

    return a + b;

};

console.log(add(10, 20));
Output:
30

Single Line Arrow Function

const square = num => num * num;

console.log(square(6));
Output:
36

Arrow Function with No Parameters

const hello = () => "Welcome to ES6";

console.log(hello());
Output:
Welcome to ES6

💡 Did You Know?

Arrow Functions do not have their own this. They inherit this from their surrounding scope.

⚠ Common Mistake

  • Using Arrow Functions as object methods.
  • Forgetting to use parentheses when returning an object.
  • Using Arrow Functions where this is required.

📥 Rest Parameters (...)

The Rest Operator (...) collects multiple values into a single array.

Example

function total(...numbers) {

    let sum = 0;

    for(let num of numbers){

        sum += num;

    }

    return sum;

}

console.log(total(10,20,30,40));
Output:
100

💡 Remember

Rest Operator collects values into an array.

📤 Spread Operator (...)

The Spread Operator (...) expands an array or object into individual elements.

Example with Array

const num1 = [1,2,3];

const num2 = [...num1,4,5];

console.log(num2);
Output:
[1, 2, 3, 4, 5]

Example with Function

const numbers = [10,20,30];

console.log(Math.max(...numbers));
Output:
30

💡 Rest vs Spread

Rest (...) Spread (...)
Collects values Expands values
Used in function parameters Used with arrays, objects and function calls

🎯 Array Destructuring

Array Destructuring allows you to extract values from an array and store them into variables easily.

Without Destructuring

const colors = ["Red", "Green", "Blue"];

const first = colors[0];
const second = colors[1];

console.log(first);
console.log(second);
Output:
Red
Green

With Destructuring

const colors = ["Red", "Green", "Blue"];

const [first, second] = colors;

console.log(first);
console.log(second);
Output:
Red
Green

Skipping Values

const numbers = [10,20,30,40];

const [a,,c] = numbers;

console.log(a);
console.log(c);
Output:
10
30

Using Rest Operator

const fruits = ["Apple","Mango","Orange","Banana"];

const [first,...others] = fruits;

console.log(first);
console.log(others);
Output:
Apple
["Mango","Orange","Banana"]

🎯 Object Destructuring

Object Destructuring allows you to extract object properties into variables.

Example

const student = {

    name: "Roshni",

    age: 20,

    city: "Mumbai"

};

const {name, age} = student;

console.log(name);

console.log(age);
Output:
Roshni
20

Rename Variables

const student = {

    name: "Roshni",

    city: "Mumbai"

};

const {name: studentName} = student;

console.log(studentName);
Output:
Roshni

💡 Tip

Destructuring makes your code shorter and easier to read.

✨ Enhanced Object Literals

ES6 introduced a shorter way to create objects.

Before ES6

let name = "Roshni";

let city = "Mumbai";

const student = {

    name: name,

    city: city

};

console.log(student);
Output:
{name: "Roshni", city: "Mumbai"}

ES6 Object Literal

let name = "Roshni";

let city = "Mumbai";

const student = {

    name,

    city

};

console.log(student);
Output:
{name: "Roshni", city: "Mumbai"}

📖 ES6 Glossary

ES6 (ECMAScript 2015)
The sixth major version of JavaScript released in 2015. It introduced modern features like let, const, arrow functions, classes, promises, modules, and template literals.
var
The old way of declaring variables. It has function scope, allows redeclaration, and is generally avoided in modern JavaScript.
let
A block-scoped variable introduced in ES6. It can be reassigned but cannot be redeclared within the same scope.
const
A block-scoped variable whose reference cannot be reassigned after declaration. It is recommended for values that should remain constant.
Block Scope
Variables declared with let and const exist only inside the block surrounded by curly braces { }.
Hoisting
JavaScript's behavior of moving declarations to the top of their scope before execution. Variables declared with let and const stay in the Temporal Dead Zone until initialized.
Arrow Function
A shorter syntax for writing functions using the => operator. Arrow functions do not have their own this.
Template Literal
A string enclosed in backticks (` `). It supports multi-line strings and variable interpolation using ${ }.
Default Parameter
A default value assigned to a function parameter that is used when no argument is passed.
Rest Operator (...)
Collects multiple values into a single array. Mostly used in function parameters.
Spread Operator (...)
Expands arrays or objects into individual elements. Commonly used for copying or merging arrays and objects.
Destructuring
A convenient syntax that extracts values from arrays or properties from objects into separate variables.
Enhanced Object Literal
An ES6 feature that allows shorter object creation by using property shorthand and concise method syntax.

❓ Frequently Asked Questions

What does ES6 stand for?

ES6 stands for ECMAScript 2015, the sixth version of JavaScript.

Is ES6 supported in all browsers?

Almost all modern browsers support ES6 features.

Should I use var?

No. Modern JavaScript recommends using let and const.

What is the difference between Rest and Spread?

Rest collects values into an array, while Spread expands arrays or objects.

Can I use Arrow Functions everywhere?

No. Avoid using them when you need the object's own this value.
