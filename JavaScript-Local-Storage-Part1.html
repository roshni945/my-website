📖 Introduction

Local Storage is a built-in browser feature that allows JavaScript to store data directly inside the user's browser. The stored data remains available even after refreshing the page or restarting the browser, making it useful for saving user preferences, themes, shopping carts, notes, and many other types of application data.

💾 What is Local Storage?

Local Storage is a Web Storage feature provided by modern browsers. It allows JavaScript to store data inside the user's browser so that the data remains available even after refreshing the page or closing and reopening the browser.

📘 Definition

Local Storage is a browser-based storage mechanism that stores data as key-value pairs. The stored data does not expire automatically and remains available until it is removed manually.

💡 Did You Know?

Local Storage can store approximately 5–10 MB of data, depending on the browser. Unlike cookies, the data is not sent to the server with every request.

📝 Syntax

Basic Syntax

localStorage.methodName(key, value);

Example 1: Save Data

localStorage.setItem("name", "Roshni");

✅ Output

The value "Roshni" is saved in the browser with the key name.

Example 2: Read Data

let username = localStorage.getItem("name");

console.log(username);

✅ Output

Roshni

🌍 Real-Life Example

Suppose you enable Dark Mode on a website. The website saves your preference in Local Storage. Even if you close the browser and visit the website again, the Dark Mode setting remains enabled because the data was stored inside the browser.

⭐ Key Points

  • ✅ Stores data inside the browser.
  • ✅ Data remains even after browser restart.
  • ✅ Stores data as key-value pairs.
  • ✅ Only strings are stored directly.
  • ✅ Objects and arrays require JSON.stringify().
  • ✅ Data stays until manually removed.

🌐 Browser Storage

Browser Storage is a feature that allows websites to store information directly inside the user's browser. It helps save user preferences, application data, and settings without using a database for every small piece of information.

📘 Definition

Browser Storage is a client-side storage mechanism used to store data inside the browser. JavaScript can save, retrieve, update, and remove this data whenever required.

💡 Did You Know?

Browser Storage is faster than sending data to a server because the information is stored directly on the user's device.

🌍 Real-Life Example

Websites like YouTube remember your preferred theme, language, and video settings using Browser Storage.

⚖️ Local Storage vs Session Storage

Both Local Storage and Session Storage are part of the Web Storage API, but they behave differently.

📊 Comparison Table

Feature Local Storage Session Storage
Lifetime Until manually removed Until browser tab is closed
Storage Limit 5–10 MB 5–10 MB
Available After Refresh ✅ Yes ✅ Yes
Available After Browser Restart ✅ Yes ❌ No
Use Case Dark Mode, Cart, Notes Login Session, Temporary Data

Example: Local Storage

localStorage.setItem("theme", "dark");

✅ Output

The selected theme is saved permanently until removed.

Example: Session Storage

sessionStorage.setItem("username", "Roshni");

✅ Output

The username is available only while the browser tab remains open.

⭐ Key Points

  • ✅ Local Storage keeps data permanently until removed.
  • ✅ Session Storage deletes data when the browser tab is closed.
  • ✅ Both store data as key-value pairs.
  • ✅ Both are part of the Web Storage API.
  • ✅ Local Storage is commonly used for user preferences and saved data.

💾 JavaScript localStorage.setItem()

The setItem() method is used to save data into Local Storage. Data is stored as a key-value pair.

📘 Definition

The localStorage.setItem() method stores a value inside the browser using a unique key.

💡 Did You Know?

Even after refreshing the page or restarting the browser, data saved using setItem() remains available.

📝 Syntax

Syntax

localStorage.setItem("key", "value");

Example 1: Save Name

localStorage.setItem("name", "Roshni");

✅ Output

The value "Roshni" is stored with the key name.

Example 2: Save Theme

localStorage.setItem("theme", "dark");

✅ Output

The user's preferred theme is saved.

🌍 Real-Life Example

When a user selects Dark Mode, the website stores the preference using localStorage.setItem(). The next time the user visits the website, the same theme is automatically applied.

⭐ Key Points

  • ✅ Stores data as key-value pairs.
  • ✅ Data remains after browser restart.
  • ✅ Keys should be unique.
  • ✅ Values are stored as strings.
  • ✅ Existing keys are overwritten with new values.

📥 JavaScript localStorage.getItem()

The getItem() method retrieves data from Local Storage using its key.

📘 Definition

localStorage.getItem() returns the value stored with the specified key.

💡 Did You Know?

If the key does not exist, getItem() returns null.

📝 Syntax

Syntax

localStorage.getItem("key");

Example 1: Read Name

localStorage.setItem("name", "Roshni");

let username = localStorage.getItem("name");

console.log(username);

✅ Output

Roshni

Example 2: Missing Key

console.log(localStorage.getItem("city"));

✅ Output

null

🌍 Real-Life Example

When a user opens a website, JavaScript reads the saved language, theme, or username using localStorage.getItem() and displays the same settings automatically.

⭐ Key Points

  • ✅ Retrieves data using its key.
  • ✅ Returns the stored value as a string.
  • ✅ Returns null if the key is not found.
  • ✅ Often used with setItem().
  • ✅ Commonly used to restore saved user preferences.

🗑️ JavaScript localStorage.removeItem()

The removeItem() method removes a specific item from Local Storage using its key.

📘 Definition

localStorage.removeItem() deletes the key-value pair associated with the specified key from the browser's Local Storage.

💡 Did You Know?

Only the specified key is removed. All other stored data remains unchanged.

📝 Syntax

Syntax

localStorage.removeItem("key");

Example 1: Remove Name

localStorage.setItem("name", "Roshni");

localStorage.removeItem("name");

✅ Output

The key name and its value are removed from Local Storage.

Example 2: Verify Removal

localStorage.removeItem("theme");

console.log(localStorage.getItem("theme"));

✅ Output

null

🌍 Real-Life Example

When a user logs out of a website, JavaScript can remove the saved login information or authentication token using removeItem().

⭐ Key Points

  • ✅ Removes only one key-value pair.
  • ✅ Other stored data remains safe.
  • ✅ Uses the key name to delete data.
  • ✅ Frequently used during logout operations.
  • ✅ Deleted data cannot be recovered unless saved again.

🧹 JavaScript localStorage.clear()

The clear() method removes all data stored in Local Storage for the current website.

📘 Definition

localStorage.clear() deletes every key-value pair stored in Local Storage for the current domain.

💡 Did You Know?

Be careful while using clear() because it removes all Local Storage data for that website.

📝 Syntax

Syntax

localStorage.clear();

Example: Clear All Data

localStorage.setItem("name", "Roshni");
localStorage.setItem("theme", "dark");

localStorage.clear();

✅ Output

All Local Storage data for the website is removed.

🌍 Real-Life Example

A website may provide a "Reset Settings" button that clears all saved preferences using localStorage.clear().

⭐ Key Points

  • ✅ Deletes all Local Storage data.
  • ✅ Does not require a key.
  • ✅ Use carefully because all saved data is removed.
  • ✅ Commonly used for reset or factory reset features.
  • ✅ Affects only the current website's Local Storage.

📦 JavaScript JSON.stringify()

Local Storage can store only strings. If you want to save an object or an array, you must first convert it into a JSON string using JSON.stringify().

📘 Definition

JSON.stringify() converts a JavaScript object or array into a JSON string so it can be stored in Local Storage or sent to a server.

💡 Did You Know?

If you try to store an object directly in Local Storage, it will be saved as [object Object], which is not useful.

📝 Syntax

Syntax

JSON.stringify(object);

Example 1: Store an Object

const user = {

    name: "Roshni",
    age: 20,
    city: "Mumbai"

};

localStorage.setItem(
    "user",
    JSON.stringify(user)
);

✅ Output

The object is converted into a JSON string and stored successfully.

Example 2: Store an Array

const colors = [

    "Red",
    "Blue",
    "Green"

];

localStorage.setItem(
    "colors",
    JSON.stringify(colors)
);

✅ Output

The array is stored as a JSON string.

🌍 Real-Life Example

An online shopping website stores the user's shopping cart as an object. Before saving it in Local Storage, JavaScript converts it into a JSON string using JSON.stringify().

📦 JavaScript JSON.parse()

After reading data from Local Storage, the JSON string must be converted back into a JavaScript object or array using JSON.parse().

📘 Definition

JSON.parse() converts a JSON string back into a JavaScript object or array.

💡 Did You Know?

Without JSON.parse(), you cannot access object properties like user.name.

📝 Syntax

Syntax

JSON.parse(jsonString);

Example 1: Read an Object

const user = JSON.parse(
    localStorage.getItem("user")
);

console.log(user.name);

✅ Output

Roshni

Example 2: Read an Array

const colors = JSON.parse(
    localStorage.getItem("colors")
);

console.log(colors[0]);

✅ Output

Red

🌍 Real-Life Example

A Notes App saves all notes in Local Storage. When the app opens again, it uses JSON.parse() to convert the stored JSON string back into an array of notes.

⭐ Key Points

  • ✅ JSON.stringify() converts objects/arrays into strings.
  • ✅ JSON.parse() converts JSON strings back into objects/arrays.
  • ✅ Local Storage stores only strings.
  • ✅ Use both methods together when storing complex data.
  • ✅ Essential for real-world JavaScript applications.

👤 Store Objects in Local Storage

Since Local Storage stores only strings, JavaScript objects must first be converted into JSON format using JSON.stringify(). Later, they can be converted back into objects using JSON.parse().

Example: Store an Object

const student = {

    name: "Roshni",
    age: 20,
    course: "BCA"

};

localStorage.setItem(
    "student",
    JSON.stringify(student)
);

Retrieve the Object

const student = JSON.parse(
    localStorage.getItem("student")
);

console.log(student.name);
console.log(student.course);

✅ Output

Roshni
BCA

📋 Store Arrays in Local Storage

Arrays must also be converted into JSON strings before saving.

Example: Store an Array

const subjects = [

    "HTML",
    "CSS",
    "JavaScript",
    "React"

];

localStorage.setItem(
    "subjects",
    JSON.stringify(subjects)
);

Retrieve the Array

const subjects = JSON.parse(
    localStorage.getItem("subjects")
);

console.log(subjects[2]);

✅ Output

JavaScript

❓ Frequently Asked Questions

1. Is Local Storage permanent?

Yes. The data remains until it is removed manually.

2. Can Local Storage store objects?

Not directly. Use JSON.stringify() before saving.

3. Is Local Storage secure?

No. Avoid storing passwords, authentication tokens, or sensitive information.

4. What is the storage limit?

Most browsers allow approximately 5–10 MB per domain.

5. What is the difference between Local Storage and Session Storage?

Local Storage persists until removed, while Session Storage is cleared when the browser tab is closed.

📖 Glossary

  • Local Storage: Browser storage that keeps data permanently.
  • Session Storage: Temporary browser storage.
  • Key: The unique identifier for stored data.
  • Value: The data associated with a key.
  • JSON: JavaScript Object Notation.
  • JSON.stringify(): Converts objects into JSON strings.
  • JSON.parse(): Converts JSON strings back into JavaScript objects.
