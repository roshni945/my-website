📖 Introduction

Asynchronous JavaScript is an important concept that allows websites to perform tasks like API calls, timers, and data loading without freezing the webpage. Understanding asynchronous programming helps developers build fast and interactive applications.

⚡ What is Asynchronous JavaScript?

JavaScript is a single-threaded programming language, which means it executes one task at a time. However, JavaScript can perform long-running tasks like API calls, timers, and file operations without blocking the main program using Asynchronous JavaScript.

📘 Definition

Asynchronous JavaScript allows JavaScript to execute tasks in the background and continue running other code without waiting for the previous task to complete.

💡 Did You Know?

Although JavaScript is single-threaded, browsers provide Web APIs that help JavaScript handle asynchronous operations.

📝 Example: Normal Synchronous Code

Synchronous Example



console.log("Start");


console.log("Middle");


console.log("End");

✅ Output

Start
Middle
End

⏳ Example: Asynchronous Code

setTimeout Example



console.log("Start");


setTimeout(function(){

    console.log("Task Completed");

},2000);



console.log("End");

✅ Output

Start
End
Task Completed

🌍 Real-Life Example

Imagine ordering food online:

  • 🍔 Place order → JavaScript starts request
  • 🚚 Delivery process runs in background
  • 📱 You can continue using the app
  • ✅ When food is ready, you get notification

⭐ Key Points

  • ✅ JavaScript is single-threaded.
  • ✅ Asynchronous code does not block execution.
  • ✅ setTimeout(), API calls, and Promises are asynchronous.
  • ✅ Event Loop manages asynchronous tasks.
  • ✅ Async programming improves website performance.

📚 JavaScript Call Stack

The Call Stack is a mechanism that JavaScript uses to keep track of function execution. It follows the LIFO (Last In First Out) principle.

📘 Definition

The Call Stack is a memory structure that stores function calls and executes them one by one.

💡 Did You Know?

JavaScript has a single Call Stack, which means it can execute only one task at a time.

📝 Example: Call Stack

Function Execution




function first(){

    console.log("First Function");

}


function second(){

    first();

    console.log("Second Function");

}


second();

✅ Output

First Function
Second Function

🌍 Real-Life Example

Think of a stack of plates:

  • 🍽️ Last plate placed → Removed first
  • 📚 Last function called → Executed first

🌐 Web APIs in JavaScript

Web APIs are provided by the browser to perform tasks that JavaScript cannot handle directly, such as timers, API requests, and DOM events.

📘 Definition

Web APIs are browser features that allow JavaScript to perform asynchronous operations outside the Call Stack.

Examples of Web APIs

  • ⏰ setTimeout()
  • 🌐 Fetch API
  • 🖱️ DOM Events
  • 📍 Geolocation
  • 💾 Local Storage

Example: setTimeout()




console.log("Start");


setTimeout(function(){


console.log("Hello");


},2000);



console.log("End");

✅ Output

Start
End
Hello

📋 Callback Queue

The Callback Queue stores completed asynchronous tasks waiting to be executed by the Call Stack.

📘 Definition

Callback Queue is a queue where callback functions wait until the Call Stack becomes empty.

🌍 Real-Life Example

Imagine a restaurant:

  • 👨‍🍳 Order completed → Added to waiting list
  • 📋 Free chef → Takes next order
  • ⚙️ Call Stack → Chef working

🔄 JavaScript Event Loop

The Event Loop is the mechanism that continuously checks whether the Call Stack is empty. If the stack is empty, it moves tasks from the Callback Queue to the Call Stack.

📘 Definition

The Event Loop manages asynchronous JavaScript execution by connecting the Call Stack and Callback Queue.

⚙️ Event Loop Working Process




1. JavaScript code enters Call Stack


            ↓


2. Async task goes to Web API


            ↓


3. Task completes


            ↓


4. Callback moves to Callback Queue


            ↓


5. Event Loop checks Call Stack


            ↓


6. Callback executes

⭐ Key Points

  • ✅ Call Stack executes JavaScript code.
  • ✅ Web APIs handle asynchronous tasks.
  • ✅ Callback Queue stores completed callbacks.
  • ✅ Event Loop connects Queue and Stack.
  • ✅ This process makes JavaScript non-blocking.

📞 Callback Functions

A Callback Function is a function that is passed as an argument to another function and executed later after a task is completed.

📘 Definition

A callback is a function that is passed into another function as a parameter and called when a specific operation is finished.

💡 Did You Know?

Callbacks are one of the oldest ways to handle asynchronous operations in JavaScript before Promises and Async/Await were introduced.

📝 Syntax

Callback Syntax



function firstFunction(callback){

    callback();

}


firstFunction(secondFunction);

Example: Callback Function

Simple Callback Example



function greet(name, callback){


    console.log("Hello " + name);


    callback();


}



function message(){


    console.log("Welcome to JavaScript");


}



greet("Roshni", message);

✅ Output

Hello Roshni
Welcome to JavaScript

🌍 Real-Life Example

When you upload a photo:

  • 📤 Upload starts
  • ⏳ Waiting for completion
  • ✅ After upload finishes, success message appears

The success message works like a callback function.

🔥 Callback Hell

When multiple asynchronous tasks depend on each other, callbacks can become deeply nested. This situation is called Callback Hell.

📘 Definition

Callback Hell is a situation where many nested callbacks make code difficult to read, understand, and maintain.

❌ Example of Callback Hell

Nested Callbacks




login(function(){


    getUser(function(){


        getData(function(){


            showResult(function(){


                console.log("Completed");


            });


        });


    });


});

⚠️ Problems of Callback Hell

  • Code becomes difficult to read.
  • Debugging becomes harder.
  • Maintenance becomes difficult.
  • Creates pyramid-like structure.

🤝 JavaScript Promises

Promises are a modern way to handle asynchronous operations. They solve the problems created by callback hell.

📘 Definition

A Promise is an object that represents the future result of an asynchronous operation.

Promise States




        Promise


           |

   -----------------

   |       |       |

Pending Fulfilled Rejected
  • ⏳ Pending: Operation is running
  • ✅ Fulfilled: Operation completed successfully
  • ❌ Rejected: Operation failed

📝 Promise Syntax

Create Promise




let promise = new Promise(function(resolve, reject){


    let success = true;


    if(success){

        resolve("Task Completed");

    }

    else{

        reject("Task Failed");

    }


});

Using then() and catch()

Promise Handling




promise

.then(function(result){

    console.log(result);

})


.catch(function(error){

    console.log(error);

});

✅ Output

Task Completed

⭐ Key Points

  • ✅ Callback functions handle tasks after completion.
  • ✅ Too many nested callbacks create Callback Hell.
  • ✅ Promises provide cleaner asynchronous code.
  • ✅ Promise has three states: Pending, Fulfilled, Rejected.
  • ✅ Promises use then(), catch(), and finally().

⚡ JavaScript Promise Methods

Promises provide different methods to handle asynchronous operations and manage multiple tasks efficiently.

📘 Definition

Promise methods are built-in methods that help us work with successful and failed asynchronous operations.

⭐ Important Promise Methods

Method Description
then() Runs when Promise is successful
catch() Handles errors
finally() Runs after completion (success or failure)
Promise.all() Runs multiple promises together
Promise.race() Returns first completed promise

📝 Example: finally()

Promise finally Example




let task = new Promise(function(resolve,reject){


    resolve("Data Loaded");


});



task

.then(function(result){

    console.log(result);

})


.catch(function(error){

    console.log(error);

})


.finally(function(){

    console.log("Task Finished");

});

✅ Output

Data Loaded
Task Finished

⏳ Async/Await in JavaScript

Async/Await is a modern way to write asynchronous code. It makes Promise-based code easier to read and understand.

📘 Definition

async makes a function return a Promise, while await pauses the execution until the Promise is completed.

💡 Did You Know?

Async/Await was introduced in ES2017 and is widely used in modern JavaScript applications.

📝 Async Function Syntax




async function functionName(){


    let result = await promise;


}

Example: Async/Await

Fetch Data Example




function getData(){


return new Promise(function(resolve){


setTimeout(function(){


resolve("Data Received");


},2000);


});


}




async function displayData(){


let result = await getData();


console.log(result);


}



displayData();

✅ Output

Data Received

🌍 Real-Life Example

When a shopping website loads products:

  • 🛒 Request product data
  • ⏳ Wait for server response
  • 📦 Display products after data arrives

Async/Await makes this process easier to write.

⚠️ Error Handling in Async JavaScript

Errors can happen during asynchronous operations. JavaScript provides error handling methods to manage these problems.

📘 Definition

Error handling allows developers to detect and handle errors without stopping the entire program.

try...catch with Async/Await

Error Handling Example




async function getUser(){


try{


let response = await fetch("invalid-url");


console.log(response);


}


catch(error){


console.log("Error Found");


}


}



getUser();

✅ Output

Error Found

⭐ Key Points

  • ✅ Async/Await makes Promise code simple.
  • ✅ async function always returns a Promise.
  • ✅ await waits for Promise completion.
  • ✅ try...catch handles asynchronous errors.
  • ✅ Modern JavaScript uses Async/Await frequently.

🔄 JavaScript Event Loop Deep Explanation

The Event Loop is one of the most important concepts in Asynchronous JavaScript. It allows JavaScript to perform non-blocking operations even though it has a single Call Stack.

📘 Definition

The Event Loop continuously checks the Call Stack and Task Queues. When the Call Stack is empty, it moves waiting tasks into the stack for execution.

💡 Did You Know?

The Event Loop is the reason JavaScript can handle timers, API requests, and user interactions smoothly without freezing the webpage.

⚙️ How Event Loop Works




JavaScript Code

       ↓

Call Stack

       ↓

Web APIs

       ↓

Callback Queue

       ↓

Event Loop

       ↓

Call Stack

⭐ Event Loop Steps

  • 1️⃣ JavaScript code enters the Call Stack.
  • 2️⃣ Asynchronous tasks are sent to Web APIs.
  • 3️⃣ After completion, callbacks move to a queue.
  • 4️⃣ Event Loop checks if Call Stack is empty.
  • 5️⃣ Callback is moved to Call Stack and executed.

⚡ Microtask Queue

The Microtask Queue stores high-priority asynchronous tasks. Promises and Async/Await use the Microtask Queue.

📘 Definition

The Microtask Queue is a queue that stores callbacks from Promises and executes them before normal callback tasks.

Examples of Microtasks

  • 🤝 Promise.then()
  • ⏳ Promise.catch()
  • ⚡ Async/Await
  • 🔄 MutationObserver

Promise Example




console.log("Start");



Promise.resolve()

.then(function(){

console.log("Promise Task");

});



console.log("End");

✅ Output

Start
End
Promise Task

⏰ Macrotask Queue

The Macrotask Queue stores normal asynchronous tasks like timers and browser events.

📘 Definition

The Macrotask Queue contains callbacks from setTimeout(), setInterval(), and DOM events.

Examples of Macrotasks

  • ⏰ setTimeout()
  • 🔁 setInterval()
  • 🖱️ Click Events
  • ⌨️ Keyboard Events

setTimeout Example




console.log("Start");



setTimeout(function(){


console.log("Timer");


},0);



console.log("End");

✅ Output

Start
End
Timer

📌 JavaScript Execution Order

When JavaScript runs asynchronous code, Microtasks have higher priority than Macrotasks.

📝 Example: Execution Priority

Microtask vs Macrotask




console.log("1");



setTimeout(()=>{


console.log("2");


},0);



Promise.resolve()

.then(()=>{


console.log("3");


});



console.log("4");

✅ Output

1
4
3
2

🌍 Real-Life Example

Think of priority system:

  • 🚨 Emergency work → Microtask Queue
  • 📋 Normal work → Macrotask Queue
  • 👨‍💻 Manager → Event Loop

⭐ Key Points

  • ✅ Event Loop manages asynchronous execution.
  • ✅ Promise tasks go to Microtask Queue.
  • ✅ setTimeout tasks go to Macrotask Queue.
  • ✅ Microtasks execute before Macrotasks.
  • ✅ Understanding Event Loop is important for interviews.

🌐 JavaScript Fetch API

The Fetch API is a modern JavaScript feature used to make HTTP requests and communicate with servers. It allows developers to get data from APIs and display it on webpages.

📘 Definition

Fetch API provides a simple way to make network requests using the fetch() method. It returns a Promise that resolves with the response from the server.

💡 Did You Know?

The Fetch API replaced older techniques like XMLHttpRequest (XHR) and is commonly used in modern web applications.

📝 Fetch Syntax

Basic Fetch Syntax




fetch("URL")

.then(response => {

    return response.json();

})

.then(data => {

    console.log(data);

})

.catch(error => {

    console.log(error);

});

📡 HTTP Requests in JavaScript

Fetch API is used to send and receive data using different HTTP methods.

Method Purpose
GET Fetch data from server
POST Send new data to server
PUT Update existing data
DELETE Remove data

🌍 Real-Life Example

When you open a weather app:

  • 📱 App sends request to weather API
  • 🌐 Server sends weather data
  • ☁️ JavaScript displays temperature

📥 GET Request using Fetch

A GET request is used to retrieve data from a server.

Fetch Data Example




fetch("https://jsonplaceholder.typicode.com/users")


.then(response => response.json())


.then(data => {


console.log(data);


})


.catch(error => {


console.log(error);


});

✅ Output

User data displayed from API.

💾 JavaScript Browser Storage

Browser Storage allows websites to store data inside the user's browser. It helps save information even after refreshing or closing the webpage.

📘 Definition

Browser Storage is a feature that allows JavaScript applications to store and retrieve data directly from the browser.

Types of Browser Storage

Storage Description
localStorage Stores data permanently until manually removed
sessionStorage Stores data only during current browser session

📦 JavaScript localStorage

localStorage allows developers to store data permanently in the browser. Data remains available even after closing the browser.

📘 Definition

localStorage stores data as key-value pairs and provides methods to save, read, update, and remove data.

⭐ localStorage Methods

Method Purpose
setItem() Store data
getItem() Retrieve data
removeItem() Remove specific data
clear() Remove all data

"name",
"Roshni"
);

Data saved in browser storage.

📖 Example: Get Data

getItem()

console.log(username);

🗑️ Example: Remove Data




localStorage.removeItem("name");

🧹 Example: Clear Storage




localStorage.clear();

sessionStorage works similar to localStorage, but the data exists only until the browser tab is closed.
