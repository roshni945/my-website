🏛️ JavaScript Classes

A Class is a blueprint used to create objects. It allows you to organize data and methods together. Classes were introduced in ES6 (ECMAScript 2015) to make object-oriented programming easier and cleaner.

💡 Did You Know?

JavaScript classes are built on top of prototypes. They provide a cleaner syntax but still use JavaScript's prototype-based inheritance internally.

✅ Basic Class Syntax

class Student {

    study(){

        console.log("Student is studying");

    }

}

const s1 = new Student();

s1.study();
Output:

Student is studying

📖 Explanation

  • class Student creates a new class.
  • study() is a method of the class.
  • new Student() creates an object.
  • The object can call the method using dot (.) notation.

🌍 Real-Life Example

class Mobile{

    call(){

        console.log("Calling...");

    }

    message(){

        console.log("Sending Message");

    }

}

const phone = new Mobile();

phone.call();

phone.message();
Output:

Calling...
Sending Message

💡 Tip

Use PascalCase for class names like Student, Employee, and Car.

🔨 Constructor Method

A constructor() is a special method that automatically runs when an object is created. It is mainly used to initialize object properties.

✅ Constructor Syntax

class Student{

    constructor(){

        console.log("Object Created");

    }

}

const s1 = new Student();
Output:

Object Created

✅ Constructor with Parameters

class Student{

    constructor(name,age){

        this.name = name;

        this.age = age;

    }

}

const s1 = new Student("Roshni",19);

console.log(s1.name);

console.log(s1.age);
Output:

Roshni
19

📖 Explanation

  • constructor() runs automatically.
  • this refers to the current object.
  • this.name stores the name.
  • this.age stores the age.

🌍 Real-Life Example

class Employee{

    constructor(name,salary){

        this.name = name;

        this.salary = salary;

    }

}

const emp = new Employee("Amit",45000);

console.log(emp.name);

console.log(emp.salary);
Output:

Amit
45000

⭐ Key Points

  • A class is a blueprint for creating objects.
  • Objects are created using the new keyword.
  • A constructor executes automatically.
  • this represents the current object.
  • One class can create multiple objects.

👨‍👦 Inheritance

Inheritance allows one class to inherit properties and methods from another class. It helps in reusing existing code and creating parent-child relationships.

💡 Did You Know?

The extends keyword is used to create a child class from a parent class.

Example:

class Person {

    constructor(name) {
        this.name = name;
    }

    greet() {
        console.log("Hello " + this.name);
    }

}

class Student extends Person {

    study() {
        console.log(this.name + " is studying.");
    }

}

const s1 = new Student("Roshni");

s1.greet();
s1.study();
Output:
Hello Roshni
Roshni is studying.

⭐ Key Points

  • extends creates inheritance.
  • Child class gets all properties and methods of parent class.
  • Reduces duplicate code.
  • Makes programs easier to maintain.

⭐ super Keyword

The super keyword calls the constructor or methods of the parent class. It must be called before using this inside the child constructor.

Example:

class Person {

    constructor(name) {
        this.name = name;
    }

}

class Student extends Person {

    constructor(name, course) {

        super(name);

        this.course = course;

    }

    details() {

        console.log(this.name + " studies " + this.course);

    }

}

const s1 = new Student("Roshni", "BCA");

s1.details();
Output:
Roshni studies BCA

💡 Quick Tip

Always call super() before using this inside the child constructor. Otherwise JavaScript will throw an error.

💻 Try It Yourself

Create an Animal class and inherit it into a Dog class.

class Animal {

    constructor(name){
        this.name = name;
    }

}

class Dog extends Animal {

    bark(){

        console.log(this.name + " says Woof!");

    }

}

const d1 = new Dog("Tommy");

d1.bark();
Output:
Tommy says Woof!

⭐ super Keyword

The super keyword is used inside a child class to access the parent class constructor and methods.

💡 Did You Know?

You must call super() before using this inside a child class constructor.

Syntax

class Parent {
    constructor(name){
        this.name = name;
    }
}

class Child extends Parent{
    constructor(name, age){
        super(name);
        this.age = age;
    }
}

Example 1

class Animal {

    constructor(name){
        this.name = name;
    }

    sound(){
        console.log("Animal makes sound");
    }

}

class Dog extends Animal{

    constructor(name){
        super(name);
    }

    sound(){
        super.sound();
        console.log(this.name + " barks");
    }

}

const d1 = new Dog("Tommy");

d1.sound();
Output:
Animal makes sound
Tommy barks

✅ Uses of super

  • Call parent constructor.
  • Access parent methods.
  • Avoid duplicate code.
  • Useful in inheritance.

⚠️ Common Mistakes

  • Using this before super().
  • Calling super() in a class without extends.
  • Forgetting to call the parent constructor.

🔒 Static Methods

Static methods belong to the class itself instead of its objects. They are called using the class name.

💡 Did You Know?

You don't need to create an object to call a static method.

Syntax

class ClassName{

    static methodName(){

    }

}

Example 1

class MathUtility{

    static square(num){
        return num * num;
    }

}

console.log(MathUtility.square(8));
Output:
64

Example 2

class Student{

    constructor(name){
        this.name = name;
    }

    static college(){

        return "Mumbai University";

    }

}

console.log(Student.college());
Output:
Mumbai University

⭐ Key Points

  • Static methods belong to the class.
  • Call them using ClassName.method().
  • Cannot be called by objects.
  • Useful for utility/helper functions.
Normal Method Static Method
Belongs to object Belongs to class
Needs object No object required
Called using object Called using class name
Uses instance data Utility/helper methods

🎯 Getters & Setters

Getters and Setters allow you to safely read and update object properties. They make your code cleaner and help control how values are accessed.

💡 Did You Know?

A getter is called like a property, not like a function.

Getter Example

class Student{

    constructor(name){
        this._name = name;
    }

    get name(){
        return this._name;
    }

}

const s1 = new Student("Roshni");

console.log(s1.name);
Output:
Roshni

Setter Example

class Student{

    constructor(name){
        this._name = name;
    }

    get name(){
        return this._name;
    }

    set name(value){
        this._name = value;
    }

}

const s1 = new Student("Roshni");

s1.name = "Shreya";

console.log(s1.name);
Output:
Shreya

✅ Advantages

  • Better data protection.
  • Cleaner syntax.
  • Add validation easily.
  • Improves code readability.

⚠️ Common Mistakes

  • Writing getter like obj.name().
  • Using the same property inside getter causing recursion.
  • Forgetting the get or set keyword.

📦 ES6 Modules (import & export)

ES6 Modules allow JavaScript code to be divided into multiple files. This makes large projects organized and reusable.

💡 Did You Know?

Modern JavaScript frameworks like React, Angular and Vue use ES6 Modules.

Step 1: Export

// math.js

export const PI = 3.14;

export function add(a,b){
    return a+b;
}

Step 2: Import

// app.js

import { PI, add } from "./math.js";

console.log(PI);

console.log(add(10,20));
Output:
3.14
30

Default Export

// message.js

export default function(){

    console.log("Welcome!");

}

Default Import

import message from "./message.js";

message();
Output:
Welcome!
Keyword Purpose
export Share variables, functions or classes.
default Export one main value.
import Use exported members.

⭐ Key Points

  • Modules split code into files.
  • Use export to share code.
  • Use import to reuse code.
  • Improves maintainability.
  • Essential for modern JavaScript development.

🔑 Symbol

Symbol is a primitive data type introduced in ES6. Every Symbol value is unique, even if two Symbols have the same description. Symbols are mainly used to create unique object property keys.

Example:

const id1 = Symbol("id");

const id2 = Symbol("id");

console.log(id1 === id2);
Output:
false

Using Symbol as Object Key:

const userId = Symbol("userId");

const user = {
    name: "Roshni",
    [userId]: 101
};

console.log(user.name);

console.log(user[userId]);
Output:
Roshni
101

💡 Did You Know?

Symbol properties do not appear in normal for...in loops or Object.keys(). They help prevent property name conflicts.

⭐ Key Points

  • Every Symbol is unique.
  • Symbols are immutable primitive values.
  • Used as unique object property keys.
  • Cannot be created using new Symbol().
  • Hidden from normal object iteration.

📚 JavaScript Set

A Set is a special JavaScript object that stores unique values only. Duplicate values are automatically removed.

💡 Did You Know?

A Set can store numbers, strings, objects, arrays, and even functions.

Example 1: Creating a Set

const numbers = new Set([10,20,30,20,10,40]);

console.log(numbers);
Output:
Set(4) {10,20,30,40}

Example 2: add()

const fruits = new Set();

fruits.add("Apple");

fruits.add("Mango");

fruits.add("Banana");

console.log(fruits);
Output:
Set(3) {"Apple","Mango","Banana"}

Example 3: has()

const fruits = new Set(["Apple","Mango"]);

console.log(fruits.has("Apple"));

console.log(fruits.has("Orange"));
Output:
true
false

Example 4: delete()

const numbers = new Set([10,20,30]);

numbers.delete(20);

console.log(numbers);
Output:
Set(2) {10,30}

Example 5: size

const colors = new Set(["Red","Blue","Green"]);

console.log(colors.size);
Output:
3

✅ Common Set Methods

  • add()
  • delete()
  • has()
  • clear()
  • size
  • values()
  • entries()

⭐ Key Points

  • Stores unique values.
  • Duplicate values are ignored.
  • Can store any data type.
  • Maintains insertion order.
  • Very useful for removing duplicates.

💧 WeakSet

A WeakSet is similar to Set, but it can store only objects. It holds objects weakly, allowing JavaScript to automatically remove them from memory when no longer needed.

💡 Did You Know?

WeakSet helps improve memory management because unused objects can be garbage collected automatically.

Example

const ws = new WeakSet();

let student = {

name:"Roshni"

};

ws.add(student);

console.log(ws.has(student));
Output:
true

⚠️ Common Mistakes

  • WeakSet cannot store numbers.
  • WeakSet cannot store strings.
  • WeakSet cannot be looped using for...of.
  • No size property exists.
Set WeakSet
Stores any value Stores only objects
Iterable Not iterable
Has size property No size property
Can use for...of Cannot use for...of
Objects remain until removed Unused objects are garbage collected

🗂️ JavaScript Map

A Map is a collection of key-value pairs. Unlike objects, Map allows keys of any data type such as numbers, strings, objects, and even functions.

💡 Did You Know?

Map remembers the order in which items are inserted.

Example 1: Create a Map

const student = new Map();

student.set("name","Roshni");

student.set("course","BCA");

student.set("age",19);

console.log(student);
Output:
Map(3) { "name" => "Roshni", "course" => "BCA", "age" => 19 }

Example 2: get()

const student = new Map();

student.set("name","Roshni");

console.log(student.get("name"));
Output:
Roshni

Example 3: has()

const student = new Map();

student.set("city","Mumbai");

console.log(student.has("city"));

console.log(student.has("age"));
Output:
true
false

Example 4: delete()

const student = new Map();

student.set("name","Roshni");

student.delete("name");

console.log(student);
Output:
Map(0) {}

Example 5: size

const colors = new Map();

colors.set(1,"Red");

colors.set(2,"Blue");

colors.set(3,"Green");

console.log(colors.size);
Output:
3

✅ Common Map Methods

  • set()
  • get()
  • has()
  • delete()
  • clear()
  • size
  • keys()
  • values()
  • entries()

⭐ Key Points

  • Stores key-value pairs.
  • Keys can be any data type.
  • Maintains insertion order.
  • Provides many useful methods.
  • Better than objects for dynamic key-value data.

💧 WeakMap

A WeakMap is similar to Map but stores only object keys. Unused objects are automatically removed from memory.

💡 Did You Know?

WeakMap is mainly used for private data and memory optimization.

Example

const wm = new WeakMap();

let obj = {};

wm.set(obj,"JavaScript");

console.log(wm.get(obj));
Output:
JavaScript

⚠️ Common Mistakes

  • WeakMap keys must be objects.
  • Cannot iterate over WeakMap.
  • No size property.
  • Cannot use for...of.
Map WeakMap
Stores any key Stores only object keys
Iterable Not iterable
Has size property No size property
Supports for...of Does not support for...of
Manual removal Automatic garbage collection

Unlike for...in, the for...of loop returns values directly.

❓ Optional Chaining (?.)

Optional Chaining (?.) is an ES2020 feature that safely accesses nested object properties without causing an error if a property doesn't exist.

💡 Did You Know?

Before Optional Chaining, developers had to write multiple if conditions to check whether an object existed.

Example 1: Without Optional Chaining

const student = {};

console.log(student.address.city);
Output:
❌ TypeError

Example 2: Using Optional Chaining

const student = {};

console.log(student.address?.city);
Output:
undefined

Example 3: Nested Object

const student = {

    name:"Roshni",

    address:{

        city:"Mumbai"

    }

};

console.log(student.address?.city);

console.log(student.address?.country);
Output:
Mumbai
undefined

✅ Advantages

  • Avoids runtime errors.
  • Cleaner code.
  • Perfect for API data.
  • No long if statements.

🆕 Nullish Coalescing (??)

The Nullish Coalescing Operator (??) returns the right-side value only if the left-side value is null or undefined.

💡 Did You Know?

Unlike ||, the ?? operator does not treat 0, false, or an empty string as missing values.

Example 1

let name = null;

console.log(name ?? "Guest");
Output:
Guest

Example 2

let age = 20;

console.log(age ?? 18);
Output:
20

Example 3

let score = 0;

console.log(score || 100);

console.log(score ?? 100);
Output:
100
0
Operator Returns Default When
|| false, 0, "", null, undefined
?? Only null or undefined

🌍 Real-Life Example

Suppose a user profile may or may not have a phone number. Using Optional Chaining and Nullish Coalescing together makes the code much safer. 

const user = {

    name:"Roshni"

};

console.log(user.phone?.number ?? "Phone Not Available");
Output:
Phone Not Available

⚠️ Common Mistakes

  • Confusing || with ??.
  • Using Optional Chaining where the object definitely exists.
  • Expecting ?. to create missing properties.
  • Using ?? for false or 0 values incorrectly.

⭐ Key Points

  • ?. safely accesses nested properties.
  • ?? provides a default value only for null or undefined.
  • Both operators make code cleaner and safer.
  • Very common in React, Node.js, and modern JavaScript.
  • Useful when working with APIs and JSON data.
